系统要求

Inrevo IoT系统设计用于采集、处理和转发数据。这些操作需要大量的计算资源以确保数据被及时处理。

影响资源消耗的因素​

影响资源消耗和系统性能的主要因素包括：

  • 数据点位数量

​ 数据点位越多，需要的计算和存储资源就越多。

  • 数据采集间隔

​ 数据采集间隔越短，需要的计算资源越多

  • 配置告警的数据点位数量

​ 需要额外的计算资源来判定是否应该触发警报。

  • 配置计算公式的数据点位数量

​ 根据配置的公式执行计算需要额外的计算资源。

  • 历史数据保存的时长

​ 历史数据保留周期越长，存储和访问所需的存储和计算资源就越多。

  • OPC UA服务器并发连接数

​ 并发连接越多，需要的计算资源就越多。

  • 访问此系统的web/应用程序数量

​ 应用程序越多，需要的计算资源就越多。

操作系统和CPU架构要求​

支持的操作系统包括 Windows 64位、Linux 64位和 Linux 32位。

支持的CPU架构包括 x64 (amd64)、arm64 (aarch64) 和 arm。

端口需求​

系统默认使用以下端口：

端口
51000Inrevo IOT 数据节点web服务端口
61000Inrevo IOT 驱动服务端口
53530Inrevo IOT 数据节点OPC UA 服务器端口
52000Inrevo IOT 监控节点web服务端口
53531Inrevo IOT 监控节点OPC UA 服务器端口

请确保上述端口未被其他应用程序使用，并且防火墙已配置为允许这些端口的流量。如果在部署过程中调整了默认端口，请更新防火墙设置。
